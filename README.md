# MOVIJA-Project-RW

Профессиональная модульная инфраструктура для автоматического развертывания VPN-сервиса (на базе Remnawave) с использованием Docker, Caddy (авто-SSL), PostgreSQL и Redis.

## 🌟 Особенности архитектуры
* **Модульность:** Установка логически разбита на независимые шаги (проверки, генерация пользователей, безопасность, оптимизация, деплой) в директории `modules/`.
* **Безопасность (Hardening):** Скрипт автоматически закрывает стандартный порт 22, генерирует случайный SSH-порт, отключает вход по паролю для `root` и настраивает UFW + Fail2ban.
* **Изоляция:** Панель администратора скрыта за случайно сгенерированным портом, в то время как клиентская страница подписок работает на стандартном 443 порту.
* **Оптимизация:** Автоматическое выделение SWAP-файла (2 ГБ) и тюнинг ядра (BBR, tcp_congestion_control) для максимальной скорости VPN.

## 📋 Предварительные требования
* **ОС:** Чистая установка Ubuntu 22.04 LTS или 24.04 LTS.
* **Домены:** Убедитесь, что A-записи ваших доменов (для панели и для страницы подписок) указывают на IP-адрес сервера **до начала установки**. Без этого Caddy не сможет выпустить SSL-сертификаты.
* **Cloudflare:** Для домена админ-панели обязательно выключите проксирование (серое облако / DNS only), так как панель использует нестандартный порт. Поддомен для подписок можно оставить проксированным (оранжевое облако).

## 📂 Структура проекта
* `setup.sh` — главный скрипт-оркестратор (точка входа).
* `modules/` — независимые bash-скрипты развертывания.
* `configs/` — статические конфигурации для Caddy, sysctl и безопасности.
* `docker-compose.yml` — описание инфраструктуры (Postgres, Redis, Remnawave Backend, Subscription Page, Caddy).
* `.env.example` — шаблон переменных окружения.

## 🚀 Быстрый старт

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ziz6956/MOVIJA-Project-RW.git
   cd MOVIJA-Project-RW
   ```

2. Выдайте права скрипту и запустите установку от имени root:
   ```bash
   chmod +x *.sh
   sudo ./setup.sh # если нужна установка панели
   sudo ./setup_node.sh # если нужна установка ноды
   ```
3. Следуйте интерактивным подсказкам (потребуется ввести домены и название проекта).

## ⚠️ ВАЖНО: В конце скрипт выведет таблицу со всеми сгенерированными портами, именами пользователей и паролями. Обязательно сохраните их! Стандартный доступ по порту 22 и root-паролю будет навсегда закрыт.

## 🔗 Первичная настройка подписок
После успешного старта кластера клиентская страница подписок может выдавать ошибку 502 Bad Gateway. Это нормальное поведение безопасности, так как микросервису требуется API-токен для связи с панелью:

Зайдите в вашу админ-панель по сгенерированному URL и порту (указаны в конце установки).

Перейдите в раздел Settings -> API Tokens и создайте новый токен.

Подключитесь к серверу по SSH, откройте файл .env (находится в директории проекта) и добавьте этот токен в самый конец:
   ```bash
   REMNAWAVE_API_TOKEN=ваш_сгенерированный_токен
   ```

Перезапустите кластер для применения изменений:
   ```bash
   sudo docker compose up -d
   ```

## 🛠 Управление кластером

Выполняйте команды из директории проекта:
* Посмотреть логи бэкенда панели: docker logs remnawave
* Посмотреть логи страницы подписок: docker logs remnawave-subscription-page
* Перезапустить все сервисы: docker compose restart
* Полностью остановить кластер: docker compose down

## 🚀 Развертывание Узла (VPN Node)

Для подключения нового сервера в качестве узла к вашей панели Remnawave, выполните следующие шаги на **новом** сервере (ноде):

1. **Склонируйте репозиторий:**
   ```bash
   git clone [https://github.com/ziz6956/movija-project-rw.git](https://github.com/ziz6956/movija-project-rw.git)
   cd movija-project-rw
   ```
2. **Запустите оркестратор узла:**
   ```bash
   sudo ./setup_node.sh
   ```
3. **Следуйте интерактивным подсказкам.**
   Скрипт запросит у вас:
   * IP-адрес основной панели (для настройки фаервола).
   * Секретный ключ (Node Secret), сгенерированный в панели.
   * Домены для страницы подписок и веб-кабинета.

Скрипт автоматически настроит безопасность (изменит SSH-порт, настроит UFW, Fail2ban), поднимет Docker-контейнеры узла и Caddy-сервер для маршрутизации.
